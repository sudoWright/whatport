The Flight Recorder, and a more accurate free app

The headline is the Flight Recorder (Pro): always-on background recording of what your USB-C ports are doing, so you can catch the intermittent faults that only show up when nobody's watching. The free app also reads power more accurately and surfaces a few things sooner.

## New: the Flight Recorder (Pro)

- **Always-on history.** Every few seconds WhatPort records each port's power, protocol, and link state to a local database. It runs in the background and survives restarts, so the timeline is still there when you come back. Memory stays flat no matter how long it runs.
- **Survives an overnight kill.** macOS sometimes shuts down background menu-bar apps to reclaim memory. With Launch at Login on, WhatPort now restarts itself within seconds when that happens, so recording carries on through the night. A deliberate Quit still quits.
- **"While you were away."** Open the recorder and it tells you what changed since you last looked: disconnects, faults, charging drops, protocol changes. If recording was interrupted, it says so, rather than letting the gap read as quiet.
- **Timeline, graphs, and per-event detail.** A selectable time range, a power graph across all ports, and a detail panel for any event showing the full state captured at the moment it happened.
- **Sessions.** Bookmark a stretch of recording, name it, and export it to JSON or CSV.
- **Port health.** A plain read on each port, scored by how often faults actually recur rather than by a single blip, with a clear note that this is activity data, not a hardware diagnosis. Acknowledge the counters to reset the baseline.
- **Alerts.** A notification when an overcurrent, link error, disconnect, or protocol downgrade happens.

## Improvements

- **More accurate power.** Readings now come from the SMC rather than the battery service, so per-port figures track reality more closely. The header splits power into in and out instead of one number that didn't mean much.
- **Protocol and speed sooner.** A port shows its protocol and link speed as soon as something connects, and notices when USB or DisplayPort speed changes.

## Updating from an earlier version

WhatPort's internal identifier changed in this release (the old one was a placeholder). macOS treats the update as a new app, so if you had Launch at Login switched on under an older version, you may find a leftover "WhatPort" entry in System Settings ▸ General ▸ Login Items. Remove it with the minus button. The new version manages its own.

## Install

Download below, unzip, and drag WhatPort.app to Applications, or:

```
brew upgrade --cask whatport
```

The app is signed and notarised by Apple.
